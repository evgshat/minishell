Ваша оболочка должна:
1) Не интерпретировать незакрытые кавычки или неопределенные специальные символы, такие как \ или ;.
2) Не использовать более одной глобальной переменной, подумайте об этом и будьте готовы объяснить, почему вы это делаете.
3)  
4) Иметь рабочую Историю.
5) Искать и запускать нужный исполняемый файл (на основе переменной PATH или с помощью относительный или абсолютный путь).

6) echo с опцией -n
7) cd только с относительным или абсолютным путем (добить с точками)
8) pwd без опций
9) export без опций
10) unset без опций
11) env без опций и аргументов
12) exit без опций
++++++
РЕАЛИЗОВАТЬ САМИМ
13) ' запрещает любую интерпретацию последовательности символов.
14) " запрещает интерпретацию последовательности символов, кроме $.
15) < должен перенаправлять ввод.
16) > должен перенаправлять вывод.
17) "<<" считывает ввод из текущего источника до тех пор, пока не появится строка, содержащая только де ограничитель. ему не нужно обновлять историю!
18) ">>" должен перенаправлять вывод в режиме append.
19) pipe | Выход каждой команды в конвейере соединяется с помощью pipe с входом следующей команды.
20) Переменные окружения ($, за которыми следуют символы) должны расширяться до своих значений.
21) $? должно расширяться до состояния выхода последней выполненной команды переднего плана. pipeline.
22) ctrl-C (печатает новое приглашение на новой строке) ctrl-D (выйти из оболочки) ctrl-\ (ничего не делает) должны работать как в bash.
+++++++++++
Функция readline может создавать утечки, которые не нужно исправлять.
Но будьте внимательны, ваш собственный код не должен давать утечек.
Вы должны ограничиться описанием темы.
Все, что не задано, не является обязательным.
Для каждого пункта, если у вас есть сомнения, возьмите bash в качестве справочника.
